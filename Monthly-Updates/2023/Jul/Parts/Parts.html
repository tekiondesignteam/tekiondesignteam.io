Roll on hold parts to SOR Hold from an RO
Figma Design Link
 
 
What?
When a repair order has parts that need to be special ordered, but also has on hand parts that are on hold for the customer, the parts department would like to be to roll all of the held parts over to the SOR request so they can close the repair order if the vehicle leaves. The ask is to have the ability to roll on hold parts & parts that need to be ordered to one SOR.When the vehicle comes back in and a new repair order is written, they should then be able to import both the special order and the held parts.
Why?
  • This is to assist with speed when a customer needs to take their vehicle away from a dealership before all repairs are completed. Currently they are need to manually remove items from a RO and manually re-add them to the new booking. This feature will speed up this process.⁠
  • Currently in Keyloop the Service Advisor will invoice out all the completed work, set the RO as offsite and rebook the current RO for the new date. They will also get the customer to sign a completed copy of the MPI that reports any defects with the vehicle.
  • In Pinewood the process is to create a new RO for the next date and rebook any items to the RO. They also have a signed MPI for compliance that reports any vehicle defects.
How?
  • Create an SOR from the Parts RO Sales screen
  • Create an SOR from the Parts SO screen
  • Push all none filled items to an existing open SOR
  • Push all none filled items to an newly created SOR
  • Pull all SOR items to a new Parts Repair Order or Part Sales Order

Ability to Print Part and Bin Label
Figma Design Link
 
 
What?
Ability to print part number labels from the Update Bin module in APP. Currently users can update from the part inquiry module in the APP.
Why?
  • DP (Quirk) will turn on part label printing ON the scanner before they start updating the bin for parts. User has the ability to print a part label manually if they do not turn on the print feature prior to scanning new bins. This is not a scalable approach. There is need to decouple and make them more intuitive and seamless.
How?
  1. Ability to Print Part Label
  2. Ability to print Bin Label
