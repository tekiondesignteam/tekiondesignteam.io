Parts Design Updates - November 2023
Automatic Fee Application
Figma Design Link
What?
Provision of adding multiple pricing details with their applicability date. Additionally, for each pricing detail, have the option for making it as automatic by specifying cases or either inclusions or exclusions.
Why?
We don't have the option to tag different fee details with their applicability date and pay types under one Fee Code. Along with that, we also don't have the option to add markup fee automatically when we sell a part.
How?
Ability to add multiple pricing details in the Franchise Pricing section when we add or edit a fee code. We also have a toggle for automatic fee application. When turned on, we can either choose to define inclusions or exclusion cases by specifying the customers, source codes and parts.

Bulk actions in Parts Repair Order
Figma Design Link
What?
To enable users to manage multiple actions for parts under different jobs.
Why?
Currently, users are required to perform part-level actions individually for each job. This repetitive process hinders user efficiency due to the excessive number of clicks involved. Consequently, the process becomes time-consuming and inconvenient.
How?
Implement a bulk action feature that allows parts counterpersons to select multiple parts and perform actions on them at once. This would save their time and improve efficiency.

Appointment number in RO fulfilment screens/documents
Figma Design Link
What?
To prevent part duplication after RO creation, display the appointment number alongside the RO number. This appointment number will be visible in the list view as part of the appointment column, in the detail view alongside the RO number, and in the print PDF
Why?
Following the creation of an appointment, technicians and parts counterpersons must make certain preparations before the customer's vehicle arrives at the dealership for service. Parts and service preparation become duplicated when the appointment number and the linked Repair Order number are different.
How?
Displaying the appointment number along with the repair order number provides users with visibility into the services and parts that have already been assigned for the customer's vehicle service. Users can view appointment details related to that parts repair order by clicking on the appointment number linked to the RO. This visibility will also be present in the P&A and appointment section in reverse, where the appointment number will have the RO number associated with it on the doc level.

Source code UI rework
Figma Design Link
What?
Recently the parts and inventory sub-module got a rework by improving the interface and experience of how the data is displayed. The same design language has to be followed for the source code sub-module as well.
Why?
In the current framework, there is only a single mode that contains both view/edit modes. Because of this users are getting confused when they make some changes to any of the source code.
How?
In the new framework, there are separate views for read/edit modes. Also on top of that reworked the tabular view for a few data fields on how the information is displayed. Also in the creation flow the data capture steps are bit more organized by introducing the stepper.

Parts BDC Portal
Figma Design Link
What & Why?
In an enterprise setup, there can be a Parts BDC persona ( customer care ) who takes orders from customers over call. This portal enables the parts BDC persona to have inventory visibility across all the dealerships and their warehouses. They'll also be able to create sales orders and quotes from within the app itself.
How?
With this new feature, we are launching a new app targeted towards the Parts BDC persona. This will enable them to have parts inventory across all the dealerships and their warehouses. They'll also be able to create sales orders and quotes from within the app itself.
