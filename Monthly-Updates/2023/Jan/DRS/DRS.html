DRS - January Design Updates
 
 
E-Signing Rules (Updated)
Figma File
 
 
This is feature revolves around signing of the document. It has multiple use cases which covers the 3rd party portal like R1 or signing integration. Some new features we are introducing such as editing the signature in between document signing process. This can be done multiple times and we are trying to bring a smooth user experience to enhance the whole signing process.

ZCS updates - Upload Drivers License
Figma File
 
 
To improve the experience we have added three extra fields in this section. We plan to auto-fetch information for these three fields from the uploaded driver's license. This is included in the 'My account' section, where we prompt the users to upload their driver's license if they haven't uploaded it in the ZCS process.

ZCS updates - Confirmation for Pay at Dealership
Figma File
 
 
If the user proceeds with the 'Pay at dealership' option, in the confirmation screen, we have added more emphasis to the statement that they have to make a payment of some particular amount at the dealership so that this information stays in their mind and is front and center.
If, at this stage, the user decides to make the payment online; we have added a link enabling users to go back and make an online payment.

ZCS updates - Partial Payments
Figma File
 
 
This feature enables the users to pay a certain amount out of the total payment when asked by the dealership. Users can view the dealership's message, the total paid amount, their receipts, and the due amount (new payment request from the dealership).
