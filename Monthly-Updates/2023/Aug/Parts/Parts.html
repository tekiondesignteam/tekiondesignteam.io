RRG: Parts Master Integration Report
Figma Design Link
What?
A report is produced to show contrasts between the old and new data after the Parts Master Integration happens. Changes in Part Master and Changes in Local Inventory reports are the two different types of reports that it will provide.
Why?
After integration, there is no way to determine what changes have been done and how they relate to the prior state. The goal of this report is to look at all the modifications that were made following the integration.
How?
  1. Introduction of the 'Integration Reports' tab in Parts & Inventory, which lists all the reports that have been produced so far for both Part Master and Local Inventory.
  2. Distinct sections will be included in each type of report for the various alterations made.

Parts Integration Dashboard (Internal)
Figma Design Link
What?
OEM send parts master updates via different file formats. Tekion's internal teams are responsible for timely and quality updates to parts master and local inventory. There are different stages through which the integration file goes through. This design enables the internal stakeholders to track where is the file stuck in the whole integration journey.
Why?
On account of delays in part master updates, the dealership blames Tekion as a platform for the delays. In reality, the reason can be many - OEM's delay in sending files or corrupt file sent or the file is stuck in any of the stages in the integration journey. Today, it's difficult to track what is causing the delay. This design helps in troubleshooting.
How?
By introducing a dashboard for internal stakeholders and informing them timely so that action can be taken at the right time, to prevent integration delays
