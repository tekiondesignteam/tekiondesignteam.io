Parts - April Design Updates
 
 
[Inchcape] Manual Receipt of vendor special order & managing floats
Figma Design Link
 
 
What?
  • This will enable to create Manual Receipt - Special Order for an RO
Why?
  • Bravo Auto users orders parts over the phone (or through OEM website). In many cases, the part number is not known. They share part description with the vendor and the RO reference (in some cases vehicle lic). PO is created after the part is created. In many cases part is added in the RO after PO is received.
How?
  • Ability to create a manual vendor special order for multiple ROs.
  • Part lines, quantities to be added in the respective repair orders after the special order is created & received.
  • Instead of float user will be able to create a part in manual receipt part line level, from the receiving module

[Inchcape] - Ability to return used cores to the vendor
Figma Design Link
 
 
What?
Users want the ability to return used cores on Vendor credit PO so that they get credit on a core
Why?
Today, the DP returns used core and damaged parts using a Misc PO Credit to process the refund and then manually remove the core on hand. They cannot have just a core on a Vendor Credit PO or Misc Credit PO.
How?
  • Ability to return the used core to the vendor directly from the Core Management which will add the used cores to a Vendor Credit PO
  • Ability to add used cores along with parts when you open Vendor Credit PO
