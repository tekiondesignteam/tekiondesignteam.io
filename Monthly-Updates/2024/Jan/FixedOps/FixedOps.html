Parts
Lock & log price code overrides in Parts RO sales and SO
Figma Design Link
What?
The feature aims to enhance the Parts Management System by implementing a controlled mechanism for changing price codes and selling prices, thereby ensuring transparency and traceability in the Parts Price Override Report.

Why?
  • Dealerships do not want every Parts user to be able to change price codes and parts selling price
  • In case of price code and selling price overrides, dealerships want to track them in the price override report
  • Some dealerships also want to treat price code override as a manual override and treat it with the highest priority
How?
  • Restriction on changing price codes and selling prices to specific roles.
  • Tracking and reporting of price codes and selling price overrides in the Price Override Report.
  • Option for dealerships to treat price code changes as manual overrides with the highest priority.

Service
Return RO Details for Damage and Rework
Figma Design Link
What?
Allow Service Advisors to configure which fields are mandatory when capturing Return RO Details, and configure custom reasons

Why?
For RRG dealerships, Service Advisors would like to capture the reasons for a return RO. Eg. damage caused during service, unsatisfactory work that didn’t fix the problem customer came in with. The reasons for return RO will also affect technician’s bonus pay when they are linked with a return RO.
How?
  • Add Return RO settings in Service Settings to configure which fields are mandatory when adding Return RO details.
  • Select the touchpoints where these settings will apply (eg. Check-In, Appointments, RO)
  • Add action to allow users to add Return RO details at any point during the journey

Write-off Invoices for Warranty (Recyclage)
Figma Design Link
What?
Warranty admin/ service advisors in RRG need the ability to select a separate ‘Recyclage’ account when reopening a payer (cancelling an invoice) for Warranty payers. This is not possible in our current reopening flow.

Why?
  • Users select the recyclage account when they know that the OEM will not be reimbursing an invoice (by checking the OEM warranty portal called GCM). They therefore cancel this invoice and record this against another account.
  • These cancelled invoices are not sent over to the OEM but are used for internal purposes. So it is necessary to identify which invoices are cancelled on the original payer and which are cancelled on the ‘recyclage’.
  • They do it before the credit posting information is received by the DMS so that the amounts receivable do not build up in the meantime.

How?
  • Provide the ability to select a recyclage account in the RO reopen flow
  • If this invoice is sent over to RRG, they have to cancel it on the recyclage account.
  • If the invoice is not yet sent over to RRG, they have the option of doing either. Either on the original payer itself, or on the Recyclage account.
