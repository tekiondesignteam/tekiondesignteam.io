Parts
PDF Configurator
Figma Design Link
What?
A configurator to control layout and contents of reports and PDFs that belongs to Parts department.

Why?
We are designing the PDF Configurator to address limitations in our current system where users cannot change the orientation of PDFs, lack preview options, and experience issues with unintended prints. This redesign aims to enhance user control and improve printing accuracy.

How?
  • Change the layout of the page
  • Wrap text
  • Adjust line height
  • Choose from three scale modes: Fit to page, Actual Size, and Custom
  • Select preset fields on the document

Figma Design Link
What?
Ability to add Aftermarket part as a OEM and a Brand

Why?
We are designing the Aftermarket Parts to meet the demand for aftermarket parts within Platform. Our competitors have implemented similar solutions in their DMS, prompting us to explore simplified approaches without compromising existing functionalities.

How?
  • Add aftermarket parts with a free-text brand field
  • Implement error handling to prevent duplicate brands
  • Retain existing hierarchical structures while incorporating aftermarket parts seamlessly
  • Ensure users can add aftermarket parts without directly mapping to a specific OEM, but still maintain functionality for adding new brands within the system

Activity log for linked parts
Figma Design Link
What?
When a part is superseeded, in the parts and inventory screen there is no way for the user to see the part details of the old part from the new part detail transaction screen, unless the search for that particular part from the parts list view. From a parts history point of view it is a bad experience.

Why?
Even after part supersession, there are many scenarios that the user need to refer the part transactions of the old part. With the current implementation user have to do multiple back navigations to get the transactions

How?
In the new transaction screen, user have the capability to select the old parts from a dropdown which we introduced along with the table filter section. Also going forward we will be using the activity log design which we designed for RRG.

GRN Number while receiving
Figma Design Link
What?
Ability to add GRN No. to multiple part lines while receiving parts. Also providing the ability to add GRN No. while mobile receiving.

Why?
We have to manually add GRN No. for each part line and then copy it. We have to do additional steps to fill invoice number as well. There is no way we can add GRN no. in receiving via mobile scanning.

How?
  • Add GRN No. for multiple part lines received via a bulk action.
  • For the GRN No., have the provision to copy to or copy from the Invoice No.
  • Adding functionality to add GRN while receiving parts
